# face_highlighter_1 使用文档

## 一、程序概述

**face_highlighter_1** 是一个基于 PyQt5 与 PyVista 的 **CAD 面高亮与点云可视化工具**。程序通过 **occwl** 与 **Open CASCADE (OCC)** 加载 STEP 模型，使用 **EntityMapper** 为每个面分配唯一 ID（face tag），并支持：

- 按配置文件对指定“面对”（左面–右面成对）进行高亮显示；
- 点击模型上的面，加载并显示该面对对应的**点云数据**（来自 NPZ 或 fallback TXT）；
- 根据点云中的 **offset**（预测值/真值）进行 **等值面拟合**（B-spline、圆柱/圆锥/球/周期旋转面等），并支持**批量拟合**与 **STEP 导出**（midsurf）；
- 多种**渲染模式**（全部面 / 仅当前面对 / 概览全部面对点云）及**透明面选择（STF）** 等辅助功能。

适用于：CAD 中厚壁件“面对”的标注与可视化、offset/validity 预测结果查看、等值面（如中间面）拟合与导出。

---

## 二、功能简介

| 功能 | 说明 |
|------|------|
| 加载配置 + STEP | 通过 TXT 配置文件指定多组“面对”（每行一组 face tag），自动加载同目录下同名 `.step` 并应用高亮 |
| 单独加载 STEP | 仅加载 STEP 模型，不高亮任何面 |
| 高亮图例 | 每组面对用不同颜色显示，左侧列表可勾选/取消显示、支持“反”（左右反转 + offset→1-offset） |
| 点击选面 | 左键点击模型上的面：若该面属于某“面对”，则加载该面对的点云；否则显示面 ID |
| 点云显示 | 点云以小球（Glyph）形式显示，颜色可由 Albedo 映射（offset/validity 预测或真值）按三端色条着色 |
| 等值面拟合 | 对当前面对点云按 offset 区间筛选后拟合曲面（B-spline 或参数化圆柱/圆锥/球/周期旋转面），支持“筛选显示”“拟合”“批量拟合” |
| 批量拟合与导出 | 对所有面对批量拟合等值面，可导出为 `配置名_midsurf.step`（原 solid + 所有拟合曲面） |
| B-spline 延展 | 批量拟合后，可对 type=bspline 的曲面做 UV 延展（extV 比例），再刷新显示 |
| 渲染模式 | **Model**：全部面；**Solo**：仅当前选中的左右面；**概览**：全部面 + 全部面对的点云（每面对采样显示） |
| 透明面 (STF) | 在 Model 模式下可进入“选择透明面”模式，点击面切换其是否在透明列表中，便于查看内部 |
| OBB 框线 | 当前面对的 OBB 包围盒线框，可开关与调节颜色、粗细 |

## 三、运行环境与依赖

- **Python**：建议 3.8+  
- **主要依赖**：  
  - `numpy`、`scipy`（插值、拟合）  
  - `vtk`、`pyvista`、`pyvistaqt`（三维显示）  
  - `PyQt5`（界面）  
  - `occwl`（CAD 加载、EntityMapper、Face 等）  
  - `pythonocc-core`（OCC：BRep、三角化、OBB、STEP 读写等）  
  - `matplotlib`（颜色，用于 Albedo 映射）

环境可通过项目中的 `environment.yml`（如存在）创建 conda 环境后运行。

---

## 四、启动方式

在项目目录下执行：

```bash
python face_highlighter_1.py
```

无命令行参数，启动后出现主窗口 **「CAD Face Highlighter (EntityMapper ID)」**。

## 五、数据文件说明

### 5.1 配置文件（TXT）

- **用途**：定义要高亮的“面对”列表，每行一组 face tag。  
- **格式**：每行一个或多个 face tag，用**空格、逗号或制表符**分隔；空行会被忽略。  
- **示例**：
  ```text
  1 2
  3 4
  5 6
  ```
  表示三组面对：(1,2)、(3,4)、(5,6)。程序会到**同目录**下查找同名 `.step`（例如配置文件为 `model.txt`，则找 `model.step`）。  
- **要求**：至少有一行且每行至少有一个可解析的整数 tag；程序会用这些 tag 在加载的 solid 上查找对应面并上色。

### 5.2 STEP 文件

- **位置**：与配置文件同目录，且主文件名与配置文件一致（仅扩展名为 `.step` 或 `.stp`）。  
- **单独加载**：也可通过“单独加载 STEP 文件”选择任意 `.stp`/`.step`，此时不会应用任何 TXT 高亮配置。  
- **面 ID**：程序使用 occwl 的 **EntityMapper** 对 solid 的 face 进行索引，界面与 NPZ 中使用的“face tag”即该索引（从 1 开始，与 cad_picker 等工具一致）。

### 5.3 点云数据（NPZ）

- **命名规则**：`{config_name}_{left_id}_{right_id}_result.npz`，例如配置文件名为 `model`，面对为 (1, 2)，则查找 `model_1_2_result.npz`。  
- **若不存在**：会尝试 `model_2_1_result.npz`（左右顺序互换）；若存在则自动按 (2,1) 作为当前面对并加载。  
- **NPZ 内容**（均为数组）：  
  - `query_points`：在 OBB 局部坐标系下的采样点（用于与 STEP 中该面对的 OBB 还原到世界坐标）；  
  - `offset_pred`、`offset_gt`：offset 预测值、真值（通常 ∈ [0,1]）；  
  - `validity_pred`、`validity_gt`：有效性（程序内 >0 视为有效，参与等值面筛选）。  
- **路径**：NPZ 需放在**配置文件所在目录**（与 STEP 同目录）；程序根据当前加载的 `config_dir`、`config_name` 及选中的面对 (left_id, right_id) 拼出路径。

### 5.4 Fallback TXT（可选）

- 当对应 NPZ 不存在时，程序会到 `config_dir/fallback` 目录下查找：  
  `{config_name}_{left_id}_{right_id}.txt`（或左右互换的命名）。  
- **格式**：需包含 `SAMPLES_DATA` 与 `Format` 的标记行，其后为多行数据，每行：`x y z offset validity`（空格分隔），其中 validity 在 TXT 中 -1 表示有效，程序会转换为与 NPZ 一致的 1 表示有效。  
- 使用 fallback 时，程序将 offset_pred=offset_gt、validity_pred=validity_gt，仅用于可视化与拟合。

## 六、界面与操作

### 6.1 工具栏

- **加载配置文件 (TXT)**：选择 TXT → 解析 face tag 组 → 自动加载同目录同名 STEP → 应用高亮；并记录 config_dir/config_name，供后续点云与批量拟合使用。  
- **单独加载 STEP 文件**：仅打开 STEP，不读 TXT，清空高亮与面对配置。  
- **清除高亮**：去掉所有面的高亮颜色，恢复默认；不卸载 STEP。

### 6.2 信息栏

窗口顶部 **lbl_info** 显示当前状态：如已加载的 STEP 名称、面数、高亮组数、当前选中的面对、点云加载结果、拟合/导出结果等。

### 6.3 左侧面板

- **高亮图例**  
  - 每组面对一行：复选框 + “反”复选框。  
  - 勾选：该组高亮显示；取消勾选：该组恢复默认颜色。  
  - “反”：对该组左右顺序取反（配置文件中的面对顺序同步更新），并对**当前已加载的该面对点云**的 offset 做 1-offset，与“取反当前面对”按钮及快捷键一致。  
  - **全选 / 取消全选**：一键勾选/取消所有组。  
  - **取反当前面对**：对当前选中的面对执行一次“反”操作（等价于找到对应行的“反”并切换）。

- **渲染模式**（下拉框）  
  - **Model (全部)**：显示所有面；默认/透明面/边线/OBB 等按全局设置。  
  - **Solo (仅当前面)**：只显示当前选中的左右两个面 + 当前面对的点云 + OBB；镜头可对准当前面对 OBB 中心。  
  - **概览 (全部面对点云)**：显示所有面 + 为每个面对加载并采样显示点云（每面对点数由代码中 `OVERVIEW_POINTS_PER_PAIR` 限制）；点击某面可切换“当前面对”并更新 OBB，但不加载单面对完整点云。

- **Model 默认**  
  - **颜色**：Model 模式下未高亮、未在透明列表中的面的默认颜色。  
  - **透明**：上述面的整体透明度（滑条 0–100 对应 0–1）。

- **透明面 (STF)**  
  - **选择透明面 (STF)**：仅在 **Model** 模式下可用。勾选后进入 STF 模式，点击面可将其加入/移出“透明面列表”；列表中的面使用“透明”滑条指定的透明度与高亮色区分。  
  - **普通/高亮**：STF 下非透明列表面 / 透明列表面的显示颜色。  
  - **透明**：透明列表内面的透明度。

- **Solo 下面颜色**  
  - 左面/右面：颜色按钮 + 透明度数值框，仅影响 **Solo** 模式下当前左右面的显示。

### 6.4 右侧面板

- **点云 Albedo**  
  - **数据源**：Offset 预测值、Offset 真值、Validity 预测值、Validity 真值。选择后点云小球按该标量在“起始–中间–结束”三端色条上映射颜色。  
  - **起始/中间/结束**：三个颜色按钮 + 中间渐变色条预览，对应标量最小、中间、最大时的颜色。

- **模型边线**  
  - **显示边线**：是否显示模型边（tube 化）。  
  - **颜色、粗细**：边线颜色与 tube 半径（相对模型包围盒对角线比例）。

- **OBB 框线**  
  - **显示 OBB 框线**：是否显示当前面对的 OBB 线框。  
  - **颜色、粗细**：OBB 线框颜色与 tube 半径。

- **点云半径**  
  - 滑条 0.1×–3×：点云小球半径相对默认的缩放。  
  - **透明**：点云整体透明度。

- **等值面拟合**  
  - **Offset**：数据源选“预测值”或“真值”。  
  - **等值**：目标 offset 值（如 0.5 表示中间面）。  
  - **Tol**：容差，筛选 offset ∈ [等值−Tol, 等值+Tol] 的点（再结合 validity>0）。  
  - **颜色、透明**：拟合出的曲面颜色与透明度。  
  - **新行为(双区间)**：勾选且等值≠0.5 时，会同时用等值与 1−等值 两个区间各做一次拟合并显示两个曲面；不勾选则只做一次（旧行为）。  
  - **筛选**：勾选后，点云只显示落在当前等值区间内的点（用于预览参与拟合的点）。  
  - **拟合**：对**当前面对**的点云按上述区间拟合等值面并显示（可多曲面）；仅影响“单次拟合”的曲面列表。  
  - **批量拟合**：对配置中**所有面对**依次加载点云、按同一等值/容差拟合，结果参与导出与延展。  
  - **导出 midsurf.step**：将当前 STEP 的 solid + 所有批量拟合得到的等值面导出为 `{config_name}_midsurf.step`，保存到 config_dir。  
  - **extV**：B-spline 曲面的 UV 延展比例（0–1）；**延展**按钮在批量拟合完成后可用，对 type=bspline 的曲面做延展并刷新显示。

### 6.5 视图与交互

- **主视图**：PyVista BackgroundPlotter，支持旋转、缩放、平移。  
- **左键点击**：  
  - 若未在 STF 模式且非概览模式：点击面 → 若该面属于某面对，则加载该面对点云并可选进入 Solo 显示；否则仅显示面 ID。  
  - 概览模式：点击面仅更新“当前面对”和 OBB，不加载单面对完整点云。  
  - STF 模式（仅 Model）：点击面切换其是否在透明列表中。

## 七、工作流程建议

1. **准备数据**：同目录下放置 `xxx.txt`（面对列表）与 `xxx.step`；每面对若有 NPZ 则放置 `xxx_left_right_result.npz`，或使用 fallback 目录下的 txt。  
2. **加载**：点击“加载配置文件 (TXT)”，确认 STEP 加载成功且高亮组数与预期一致。  
3. **查看面对**：在左侧勾选/取消高亮组；点击模型上的面，加载对应面对的点云。  
4. **点云与等值面**：在右侧选择 Albedo 数据源与等值/容差，可先勾选“筛选”查看参与拟合的点，再点“拟合”查看单面对等值面。  
5. **批量拟合与导出**：设置好等值、容差、数据源后点“批量拟合”；完成后可调节 extV 并“延展”，最后“导出 midsurf.step”。  
6. **渲染模式**：需要专注当前面对时切到 Solo；需要总览所有面对点云时切到概览。

## 八、等值面拟合说明

- **逻辑**：根据当前（或批量）面对的**左右面类型**自动选择拟合方式：  
  - 双圆柱 → 圆柱参数化曲面  
  - 双圆锥 → 圆锥参数化曲面  
  - 双球面 → 球面参数化曲面  
  - 周期旋转面等 → 周期 B-spline 或周期旋转面  
  - 其余 → 非周期 B-spline（PCA 参数化 + bisplrep）  
- **筛选**：仅使用 offset 在 [等值−Tol, 等值+Tol] 且 validity>0 的点；点数过少会跳过该面对。  
- **导出**：批量拟合得到的曲面在导出 STEP 时，能转为 OCC 曲面的会以精确曲面写入，否则用三角网格转成 shape 写入同一 compound。

## 九、快捷键

- **Ctrl+Shift+F**：对当前选中的面对执行“取反”（左右顺序反转 + offset→1−offset），与“取反当前面对”按钮一致。

## 十、可调常量（程序顶部）

在 `face_highlighter_1.py` 文件顶部可修改以下常量，以控制行为与性能：

| 常量 | 含义 | 默认 |
|------|------|------|
| `POINT_CLOUD_MAX_POINTS` | 单面对点云最大显示点数，0 表示不限制；>0 时从 NPZ 均匀采样 | 0 |
| `OVERVIEW_POINTS_PER_PAIR` | 概览模式下每个面对最多显示的点数 | 500 |
| `ALBEDO_COLOR_LOW/MID/HIGH` | 点云 Albedo 映射的三端颜色（低/中/高） | 蓝/绿/红 |
| `ISOSURFACE_FIT_N_GRID` | 等值面拟合的网格分辨率（u/v 方向） | 50 |
| `ISOSURFACE_FIT_SMOOTHING` | B-spline 拟合平滑参数，None 时自动按点数计算 | None |
| `HIGHLIGHT_COLORS` | 高亮组使用的颜色列表（十六进制字符串） | 红绿蓝黄等 20 色 |

## 十一、常见问题

- **“未找到同名 STEP 文件”**：确保 TXT 与 STEP 在同一目录，且主文件名一致（仅扩展名不同）。  
- **“面对 npz 未找到或加载失败”**：检查 NPZ 命名是否为 `{配置名}_{左id}_{右id}_result.npz`，或是否提供 fallback 目录下的 txt。  
- **等值面拟合失败**：增大容差 Tol，或确认该面对 NPZ 中在等值附近有足够多 validity>0 的点。  
- **导出 midsurf.step 灰显**：需先执行“批量拟合”，完成后按钮才会可用。  
- **延展按钮灰显**：同样需先完成“批量拟合”；延展仅对 type=bspline 的曲面生效。
